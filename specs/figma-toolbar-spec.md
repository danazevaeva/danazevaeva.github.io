# Спецификация: Figma Capture Toolbar

## Обзор

Переиспользуемый тулбар для любых сгенерированных HTML-экранов. Позволяет в один клик отправить дизайн в Figma, захватить весь экран (включая скролл), выбрать отдельный элемент или открыть файл в Figma.

**Активация:** Тулбар виден только при наличии `?dev` в URL. Без параметра — чистая страница, без следов тулбара.

---

## Интеграция

Claude при генерации любого HTML-экрана должен:

1. Добавить `<script src="/figma-toolbar.js"></script>` перед `</body>`
2. Больше ничего не нужно — тулбар сам создаёт DOM, стили и логику

---

## Функции тулбара

### 1. Отправить в Figma
- **Кнопка:** "Отправить в Figma" с иконкой Figma
- **Поведение:**
  - Инжектит `capture.js` от Figma MCP (`https://mcp.figma.com/mcp/html-to-design/capture.js`)
  - Перед захватом раскрывает скроллируемые контейнеры на полную высоту (убирает `overflow: auto/scroll/hidden`, `position: absolute`, фиксированные `height`) — чтобы вся страница попала в захват
  - Добавляет hash-параметры `#figmacapture=...` к текущему URL и перезагружает
  - После захвата восстанавливает оригинальную раскладку

### 2. Весь экран → Clipboard
- **Кнопка:** "Весь экран" с иконкой у��тройства
- **Поведение:**
  - Через `html2canvas` (CDN, scale: 2) захватывает `.device` или `body` (если `.device` нет)
  - Раскрывает скролл перед захватом, восстанавливает после
  - Копирует PNG @2x в буфер обмена (`navigator.clipboard.write`)
  - Фоллбэк: скачивает файл если clipboard API недоступен
  - Toast: "Скопировано — вставьте в Figma (⌘V)"

### 3. Выбрать элемент
- **Кнопка:** "Выбрать элемент" с иконкой курсора
- **Поведение:**
  - Переключает режим пикера (toggle)
  - При наведении: фиолетовый outline 2px #A259FF на элементе
  - Клик: захватывает выбранный элемент через html2canvas → clipboard
  - Элементы тулбара игнорируются при наведении/клике
  - Toast: "Кликните на элемент для захвата"

### 4. Открыть в Figma
- **Кнопка:** "Открыть в Figma" с иконкой ссылки
- **Поведение:**
  - Открывает URL файла Figma в новой вкладке
  - URL файла хранится как `data-figma-file` атрибут на `<script>` теге или в `window.__FIGMA_FILE_URL__`
  - Если URL не задан — кнопка скрыта

## Логика раскрытия скролла для захвата

Перед захва��ом (Figma MCP или html2canvas) тулбар:

1. Находит все элементы с `overflow: auto`, `overflow: scroll`, `overflow: hidden` внутри `.device` (или `body`)
2. Сох��аняет их оригинальные стили в массив
3. Устанавливает: `overflow: visible`, `position: static`, `height: auto`
4. Если `.device` имеет фиксированную высоту — устанавливает `height: auto`, `overflow: visible`
5. Ждёт 100ms (reflow)
6. Выполняет захват
7. Восстанавливает все стили из сохранённого массива

## Пример подключения

```html
<!-- В конце любого сгенерированного HTML, перед </body> -->
<script src="/figma-toolbar.js"></script>
```

С указанием Figma-файла:

```html
<script src="/figma-toolbar.js" data-figma-file="https://www.figma.com/design/ATZznGOIRTrNYcBGo9zdoe"></script>
```

---

## Важно

- Тулбар НЕ должен влиять на оригинальную страницу (все стили scoped, все элементы в shadow DOM или с уникальными префиксами)
- Без `?dev` в URL — файл скрипта загружается, но ничего не рендерит и не модифицирует DOM
- Раскрытие скролла — временное, только на момент захвата, с гарантированным восстановлением
